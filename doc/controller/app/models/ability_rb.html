Description

Abstracts all mapping of permissions to individual resources for OpenShift. A permission is a specific action that a user can take - for most resources the permission is granted either because of the role the user has on the resource, or due to a specific ownership relation.

Roles are defined as being a set of strictly increasing permissions on a particular resource - having a higher role entitles you to all permissions of the lower role.

The permissions a user has at any time on a resource may be limited by the set of scopes they are operating under (used by authorization tokens) so that delegated authority can be granted.
